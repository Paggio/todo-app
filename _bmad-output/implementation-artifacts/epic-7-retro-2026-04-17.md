## 2. What Went Well

### 2.1 Every carried-debt item closed

Epic 6's retro produced 11 commitments (C1–C5 process + A1–A6 technical). **All 11 landed in Epic 7.** A4 (optimistic temp-ID DELETE race) had been on the list since Epic 3 — three epic boundaries. The `id < 0` guard in `useDeleteTodo` isn't the complete fix (the residual create/delete race tail is documented in `deferred-work.md`), but the user-visible 404 is gone. First cycle with no slippage.

### 2.2 Composition Checks validated as a process

Epic 6's retro hypothesis was that per-feature ACs don't catch assembly bugs — evidence was `bfa0d62`, a post-release UI polish commit fixing popover clipping, FAB spacing, and category discoverability. Epic 7 added Composition Checks sections to both 7.1 and 7.2 Dev Notes with ten explicit cross-feature interactions to Playwright-verify before marking each story done. **Zero post-release polish commits in Epic 7.** The hypothesis holds.

### 2.3 First RTL component tests in the codebase

A1 from Epic 6's retro called for the first RTL component test. Story 7.1 delivered `view-switcher.test.tsx` (12 tests covering aria-selected, URL updates, fallback for unknown values, roving tabindex). Story 7.2 followed with `deadline-group-header.test.tsx` (8 tests) and `by-deadline-view.test.tsx` (5 tests). Frontend test count: 18 → 52. `@testing-library/user-event` was explicitly declined — `fireEvent.keyDown` covered the roving-tabindex assertions in jsdom. Zero new test dependencies.

### 2.4 Pattern reuse reached three generations

- `CategorySectionHeader` (Story 5.3) → `DeadlineGroupHeader` (7.2): structural clone, not subclass — premature abstraction correctly avoided.
- `DueThisWeekView` (7.1) → `ByDeadlineView` (7.2): parallel shape; selector, category-chip lookup, mutation wiring all mirrored.
- `selectDueThisWeek` (7.1) → `selectByDeadline` (7.2): same comparator shape, shared `PRIORITY_SORT_KEY` export.
- `TodoItem.categoryName` prop (added in 7.1) consumed unchanged in 7.2.

### 2.5 Forward-loading still paying rent

Epic 5 Story 5.1 expanded the Todo model with `priority`, `deadline`, and `category_id` in a single migration. Epic 6 consumed that expansion with zero backend work. Epic 7 also consumed it with zero backend work. Three epics served by one forward-loaded migration. The architecture rule `"Implement view filtering client-side using TanStack Query selectors — never create dedicated API endpoints per view"` held: all three views are lenses over one `["todos"]` fetch. NFR14 (< 500ms) was satisfied by memoised selectors, not new endpoints.

### 2.6 The A6 "correct reuse" call

Epic 6 retro A6 said *"reuse `formatDeadline` for DeadlineGroupHeader grouping."* The literal reading would've been to parse `formatDeadline`'s display text back into buckets — brittle coupling. Story 7.2 recognised the intent and added a sibling helper `getDeadlineBucket` that shares the SAME parse primitives (`parseDeadlineDate`, `getToday`, `daysDiff`) as `formatDeadline` rather than routing through its display-string output. Retro action items were read for *why*, not just *what*.

## 3. Challenges

### 3.1 Selector-level unit tests still absent

`selectDueThisWeek` and `selectByDeadline` are the core business logic of Epic 7 — temporal filter + multi-tier sort + fixed-order grouping — and neither has dedicated unit tests. `getDeadlineBucket` is tested in isolation (9 tests covering boundaries); the comparators and fixed-order grouping are only verified indirectly via Playwright composition checks. Explicitly flagged as Epic 7 retro follow-up in `deferred-work.md` entry for 7-2.

### 3.2 Residual items from 7.1's code review

Two items deferred from Story 7.1 code review:

- **Optimistic-create/delete race tail** (`use-todos.ts:155`) — The `id < 0` guard stops the HTTP DELETE, but `onSettled` still invalidates `["todos"]`. If a pending POST resolves after an optimistic delete, the refetch pulls the server-created todo back into the cache — "deleted" item reappears. Full fix needs aborting the pending create or chaining a real delete. Low frequency (sub-second user interaction); partial fix is still a net improvement.
- **`selectDueThisWeek` "today" boundary captured inside memo** (`use-todos.ts:471`) — `new Date()` evaluated inside the function, `useMemo` keyed only on `todos`. If the app is left open past local midnight without any mutation, the week boundary doesn't advance. Self-heals on any cache change. Low severity.

Not regressions. Documented in `deferred-work.md` with rationale. Carried forward as Epic 7 retro A2.

### 3.3 Pre-existing auth deep-link gotcha

`/?view=week` loaded unauthenticated drops the search param through the login redirect (`AuthGuard` → `/login` → `navigate(from.pathname)` reads `.pathname` only, not `.search`). Documented in Story 7.1 Debug Log as pre-existing and out of scope for 7.1; surfaced again in 7.2 Composition Checks AC #8 with the same "pre-existing, not a 7.2 regression" disposition. Epic 7 is the first consumer that cares about query-string persistence across auth, so this deserves a follow-up if deep-link sharing matters.

### 3.4 Tailwind v4 `max-[400px]:` silent no-op

Story 7.1 discovered that Tailwind v4's JIT did not emit `@media (max-width: 400px)` rules for the arbitrary `max-[400px]:` modifier in this project. Worked around by defining two dedicated CSS classes (`.view-switcher-label-short` / `.view-switcher-label-long`) gated by one `@media (max-width: 400px)` block in `index.css`. Documented only in the 7.1 Debug Log — worth lifting into `architecture.md` alongside the Popover Overflow Pattern so future stories don't rediscover it.

## 4. Key Insights

1. **Process commitments work when they land in Dev Notes, not just retro docs.** C4 (Composition Checks) and C5 (Carried Debt callout) only became effective once written into every new story's Dev Notes. "We'll remember" is not a process; "this section is mandatory in every story file" is.

2. **Pattern reuse has a ceiling — and we found it correctly.** The temptation in 7.2 was to extend `CategorySectionHeader` into a generic `CollapsibleSectionHeader`. The story explicitly declined: two concrete headers is better than one abstract one until a third case arrives.

3. **Retro action items need the *why*, not just the *what*.** A6 was interpreted correctly because the rationale was in the retro body; a literal reading would've produced a brittle fix.

4. **Zero-new-dependencies is a cultural artifact, not a rule.** Epic 4 → 7 added zero runtime deps. The streak is now quoted as precedent in Story Dev Notes. Culture eats architecture documents for breakfast.

5. **This was the smoothest epic yet.** 2 stories, 1 in-story code-review patch, 0 post-release polish commits, 11/11 previous-retro commitments closed. The process investments from Epic 5 and Epic 6 retros compounded into genuine velocity in Epic 7.
