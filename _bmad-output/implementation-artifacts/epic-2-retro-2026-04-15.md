**The action-item mechanism is proven ‚Äî items land, they are not ceremonial.**

### 2.2 TODO-marker + throw-stub discipline for cross-story handoff

Story 2.1 shipped `useLogin` / `useLogout` as `throw` stubs (loud failure on accidental use), with `TODO(Story 2.3)` markers in `core/deps.py` and `TODO(Story 2.4)` in `api.ts`. Each subsequent story replaced its marker with a one-file surgical edit and no adjacent rework. This is the cleanest cross-story handoff so far in the project.

### 2.3 Security hygiene baked in from the first auth story

- **Timing-safe login path:** `_DUMMY_PASSWORD_HASH` at cost-matched bcrypt 12 ‚Äî unknown-email and wrong-password requests have identical latency, no enumeration side-channel.
- **Identical cookie attributes on `set_cookie` / `delete_cookie`** ‚Äî the single most common logout bug (browser creates a second cookie instead of deleting) explicitly avoided.
- **Event-based 401 decoupling:** `api.ts` dispatches `CustomEvent("auth:unauthorized")` rather than importing React; `AuthProvider` subscribes via `window.addEventListener`. Framework-agnostic boundary holds.
- **Response shape enforcement:** `UserRead` response model guarantees `hashed_password` never ships; verified by shape assertions, not by inspection.
- **Isolation proven:** `test_me_isolates_to_cookie_owner` swaps cookies between two registered users and confirms `/me` scopes to the cookie owner ‚Äî trivially small test, foundational guarantee for all of Epic 3.

### 2.4 Review triage quality

30 findings across the epic were deliberately triaged ‚Äî 2 Patch, 5 Defer, 23 Decision. `[Decision]` entries document intentional choices with written rationale (stateless JWT, auth module split for `react-refresh/only-export-components`, `errors.py` at package root vs `core/`). Nothing dismissed as a one-liner. Review signal remained high.

### 2.5 Test maturation through the epic

## 3. Challenges

### 3.1 bcrypt 5.x vs passlib 1.7.4 incompatibility

First `make test-backend` in Story 2.1 failed 4/8 tests with `ValueError: password cannot be longer than 72 bytes` and `AttributeError: module 'bcrypt' has no attribute '__about__'`. Workaround: pin `bcrypt>=4.0.0,<5.0.0`. Passlib 1.7.4 is end-of-life. Tracked in `deferred-work.md`; workaround is stable but living on borrowed time.

### 3.2 Alembic autogenerate fought the naming convention

Architecture spec mandates `idx_{table}_{column}`; autogen emits SQLAlchemy default `ix_`. Story 2.1 hand-edited the migration. **This is the highest-leverage unfixed debt going into Epic 3** ‚Äî every future migration needs the same manual step unless `SQLModel.metadata.naming_convention` is configured once.

### 3.3 `users.created_at` is timezone-naive

`DateTime()` without `timezone=True` ‚Äî inserts are UTC but the column type does not enforce it. Story 3.1 will introduce `todos.created_at`; replicating the pattern without thinking would double down on the debt.

### 3.4 ESLint forced an auth module split

`react-refresh/only-export-components` rejected combining `AuthProvider` (component) + `useAuth`/`useRegister` (hooks) in one `use-auth.ts` file. Split into `auth-context.ts` / `auth-provider.tsx` / `use-auth.ts`. Architecture spec said one file; the split is a documented three-story deviation. Minor, but real.

### 3.5 `errors.py` landed at package root

Architecture spec implied `core/` for cross-cutting concerns; the pragmatic ship decision was package root. Precedent now set ‚Äî if Epic 3 adds more cross-cutting modules, we should either ratify or correct the pattern.

### 3.6 Epic 1 security cluster still outstanding

D1 (root container), D2 (unconditional `--reload`), D3 (weak `JWT_SECRET` boot refusal), D4 (weak Postgres creds), D5 (`.env` git-ignore verification) ‚Äî zero progress in Epic 2. Project lead reaffirmed the local-only defer in this retrospective (2026-04-15). Carrying forward consciously.

## 4. Key Insights

1. **Throw-stubs beat silent no-ops for forward-scaffolding.** `useLogin = () => { throw ... }` created pressure on reviewers to confirm the replacement story actually wired it up. Silent stubs would have slipped.
2. **The "same-day burst after the load-bearing first story" pattern is real.** Story 1.1 was Epic 1's shakedown; Story 2.1 was Epic 2's shakedown. Both epics finished the remaining stories in a compressed follow-up window. Expect Story 3.1 to behave similarly ‚Äî allocate accordingly.
3. **Event-based decoupling pays in tight-radius architectures.** `api.ts` stayed free of React imports throughout Epic 2 because the 401 interceptor used `window.dispatchEvent`. Epic 3's optimistic-mutation error toasts should respect the same discipline.
4. **Tests as regression fences, not coverage targets.** The 26-test count is a floor, not a goal. Epic 3 stories should grow it proportional to scope; any story that doesn't add tests needs justification.
