# Story 3.1: Todo CRUD API Endpoints

Status: done

## Story

As an authenticated user,
I want API endpoints to create, read, update, and delete my todos,
so that my task data is managed and persisted on the server.

## Acceptance Criteria

1. **Given** the Todo SQLModel (`id`, `user_id` FK, `description`, `is_completed` default `false`, `created_at`) and its Alembic migration are created **When** an authenticated user sends `GET /api/todos` **Then** the server returns an array of all todos belonging to that user, ordered by `created_at` descending (FR11)

2. **Given** an authenticated user **When** they send `POST /api/todos` with `{ "description": "Buy milk" }` **Then** the server creates a todo with `is_completed=false` and `created_at` set to current UTC time, scoped to `user_id` from JWT, and returns the created todo object (FR10, FR16)

3. **Given** an authenticated user **When** they send `POST /api/todos` with `{ "description": "" }` or missing description **Then** the server returns 422 with `{ "detail": "Description cannot be empty", "code": "VALIDATION_ERROR" }` (FR15)

4. **Given** an authenticated user owns a todo **When** they send `PATCH /api/todos/{id}` with `{ "is_completed": true }` **Then** the server updates the todo and returns the updated object (FR12, FR13)

5. **Given** an authenticated user owns a todo **When** they send `DELETE /api/todos/{id}` **Then** the server deletes the todo and returns 204 (FR14)

6. **Given** an authenticated user **When** they attempt `PATCH` or `DELETE` on a todo they don't own or that doesn't exist **Then** the server returns 404 with `{ "detail": "Todo not found", "code": "TODO_NOT_FOUND" }`

7. **Given** all todo API responses **When** they are returned **Then** JSON uses snake_case keys, dates are ISO 8601 UTC strings, booleans are `true`/`false`, and IDs are integers **And** API responses complete within 500ms under normal load (NFR3)

## Dev Notes

### Critical Architecture Constraints

- **API error format:** `{ "detail": "Human-readable message", "code": "MACHINE_READABLE_CODE" }` тАФ enforced via `api_error()` from `app/errors.py` and the custom exception handlers in `install_exception_handlers()`. Use `api_error()` for all domain errors; let Pydantic's `RequestValidationError` flow through the existing 422 handler for input validation. [Source: architecture.md#Format Patterns L296-310]
- **API routes:** Resource-based under `/api` prefix тАФ `GET /api/todos`, `POST /api/todos`, `PATCH /api/todos/{id}`, `DELETE /api/todos/{id}`. Mount the router with `prefix="/api/todos"`. [Source: architecture.md#API & Communication Patterns L196-207]
- **Per-user isolation:** All todo queries MUST filter by `user_id` from the validated JWT. Use `Depends(get_current_user)` on every endpoint and filter with `Todo.user_id == user.id`. This is the same dependency injection pattern established in `auth.py`. Never expose a 403 for not-owned resources тАФ return 404 to avoid leaking existence. [Source: architecture.md#Authentication & Security L189]
- **Naming conventions:** Python files use snake_case (`todo.py`, `todos.py`); variables/functions use snake_case; classes use PascalCase; JSON response keys use snake_case. [Source: architecture.md#Naming Patterns L252-275]
- **Response format:** Direct object or array тАФ no response wrappers (`{ data: ..., success: true }`). Dates as ISO 8601 UTC strings. Booleans as `true`/`false`. IDs as integers. [Source: architecture.md#Format Patterns L296-310]
- **Error codes for this story:** `TODO_NOT_FOUND`, `VALIDATION_ERROR` (reused from existing handler), `UNAUTHORIZED` (reused from `get_current_user`). [Source: architecture.md#Error Codes L304]
- **SQLModel is ORM:** Todo model serves as both DB table definition and the basis for response serialization. Separate Pydantic `BaseModel` classes for request schemas (`TodoCreate`, `TodoUpdate`) and response schemas (`TodoRead`). [Source: architecture.md#Data Architecture L176-180]
- **No `any` types, no hardcoded values, no response wrappers.** [Source: architecture.md#Anti-Patterns L359-366]

### Previous Story Intelligence (Epic 2, Stories 2.1-2.4)

1. **`get_current_user` dependency is production-ready.** Returns the full `User` ORM object with `user.id` for scoping. Raises 401 `UNAUTHORIZED` for missing/invalid/expired JWT. All 26 backend tests pass. Use `Depends(get_current_user)` on every todo endpoint. [Source: `backend/app/core/deps.py` L37-64]

2. **`api_error()` is the domain error constructor.** Signature: `api_error(status_code, message, code) -> ApiError`. The `install_exception_handlers` in `app/main.py` via `errors.py` coerces `ApiError` to the `{ "detail", "code" }` envelope. Pydantic `RequestValidationError` is handled separately and also coerced. [Source: `backend/app/errors.py` L1-58]

3. **`_assert_error_envelope` test helper** verifies error responses match the architecture contract: exactly two keys (`detail`, `code`), `code` matches expected, `detail` is a non-empty string. Reuse this pattern in `test_todos.py`. [Source: `backend/tests/test_auth.py` L37-48]

4. **Test fixture pattern:** `conftest.py` uses in-memory SQLite with `dependency_overrides[get_db]`. The `client` fixture is per-test hermetic. Register a user via `client.post("/api/auth/register", ...)` to get an auth cookie in the test client's cookie jar тАФ subsequent requests automatically include the cookie. [Source: `backend/tests/conftest.py` L1-56]

5. **Migration pattern:** Existing migration at `backend/alembic/versions/7e729bfa2ac7_create_users_table.py` manually renamed the index to `idx_users_email`. Task 1 configures the naming convention so the todos migration gets the correct `idx_todos_user_id` automatically.

6. **Router mounting pattern:** `app.include_router(auth_router.router, prefix="/api/auth", tags=["auth"])` in `main.py`. The todos router follows the same pattern with `prefix="/api/todos"`. [Source: `backend/app/main.py` L21]

7. **Schema co-location pattern:** Request/response schemas are defined at the top of the router file (e.g., `RegisterRequest`, `LoginRequest`, `UserRead` in `auth.py`). Follow the same pattern for `TodoCreate`, `TodoUpdate`, `TodoRead` in `todos.py`. [Source: `backend/app/routers/auth.py` L27-62]

8. **Alembic autogenerate** only detects models that have been imported before `target_metadata = SQLModel.metadata` resolves. Importing `Todo` in `models/__init__.py` and `tests/conftest.py` ensures it registers with metadata in both production and test contexts.

9. **Cookie-based auth in tests:** `TestClient` automatically persists cookies across requests. After `client.post("/api/auth/register", ...)`, subsequent `client.get("/api/todos")` includes the auth cookie. To test cross-user isolation, save `response.cookies[settings.auth_cookie_name]`, clear cookies, register a second user, then swap cookies back. [Source: `backend/tests/test_auth.py` L313-330]

10. **`users.created_at` timezone issue:** The column uses `DateTime()` without `timezone=True`. Retro T2 says to use `DateTime(timezone=True)` for `todos.created_at` from the start. Optionally fix `users.created_at` in the same migration.
